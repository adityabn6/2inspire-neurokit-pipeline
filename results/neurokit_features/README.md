# NeuroKit ECG + RSP features

Generated by `neurokit_pipeline/` (port of the MOXIE Pipeline ECG/RSP processing
adapted to 2Inspire's raw Hexoskin signals: `record_*/ECG_I.wav` @ 256 Hz;
`respiration_{thoracic,abdominal}.wav` @ 128 Hz). One local loop over all
participants × 3 pieces — no cluster. Re-run from the repo root with:

```
python neurokit_pipeline/run_all.py --participant all --session all
python neurokit_pipeline/plots.py
```

See the top-level `README.md` and `docs/` for full documentation.

## Files
| File | Rows | Description |
|------|------|-------------|
| `ecg_hrv_piece.csv` | 74 | One row / participant × piece. Whole-piece HRV from **raw-ECG** R-peaks (`nk.ecg_process` → `nk.hrv_time/frequency/nonlinear`), optional RSA (`nk.hrv_rsa`), plus QC (`ecg_quality_mean`, `pct_low_sqi`, `n_rate_outliers`). |
| `rsp_features_piece.csv` | 148 | One row / participant × piece × channel (thoracic, abdominal). `nk.rsp_process(khodadad2018)` → rate/amplitude/RVT means+SDs, slope & phase inhale/exhale ratios, `nk.rsp_rrv`, `nk.rsp_intervalrelated` (NK_ prefix). |
| `hrv_raw_vs_rri.csv` | 74 | **Cross-check.** HRV from raw ECG vs HRV from the device `RR_interval.csv` (onboard beat detection), with `delta_rmssd` / `delta_mean_hr`. |
| `ecg_continuous_1hz.csv` | — | Long: participant, piece, feature ∈ {`hr_bpm`, `hrv_rmssd`}, `t_pct` (0–100 % of piece), value. |
| `rsp_continuous_1hz.csv` | — | Long: + `channel`; feature ∈ {`rsp_rate`, `rsp_amplitude`}. |
| `failures.md` | — | Skipped sessions (P17 has no ECG/RSP/RRI on disk for any piece). |

Plots in `plots/neurokit/`: `box_{ecg,rsp}_by_piece.png` (one box per piece) and
`traj_*.png` (per-participant trajectories on a 0–100 % piece axis + mean ± 95 % CI,
one panel per piece).

## ⚠️ Raw-ECG vs device-RRI caveat
`mean_hr` matches almost exactly between sources (median |Δ| ≈ 0.06 bpm), but for
~a third of sessions **raw-ECG RMSSD is wildly inflated** relative to the device
RRI (e.g. P2/P3/P13 raw RMSSD 400–1000 ms vs device 100–500 ms). Hexoskin's
textile single-lead ECG is noisy; `nk.ecg_process` peak detection inserts/misses
beats, which barely moves mean HR but explodes the successive-difference HRV
metrics (RMSSD, pNN50, SD1, HF). **For beat-to-beat HRV, prefer the device-RRI
path** (`hrv_raw_vs_rri.csv` `rri_*`); use raw-ECG HRV only where
`delta_rmssd` is small and `ecg_quality_mean` is high. The piece-3 spread in the
HRV box plots is driven by these noisy sessions, not by a real piece effect.
